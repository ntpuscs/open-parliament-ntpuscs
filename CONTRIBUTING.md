# 貢獻指南

感謝您幫助我們維護「北大三峽議事資訊網」！為維持專案品質及其穩定運作，確保本會秘書處與專業技術人員順暢協作，請參考以下指南。

## 參與角色

### 技術維護人員

本專案最初由第26屆學生議會開發、維護。若您提供技術協助，原則上請遵守以下開發規範：

1.  請一律使用 `pnpm` 作為套件管理工具，避免使用 `npm` 或 `yarn` 造成 lock 檔衝突。
2.  請遵守 Conventional Commits 規範。其中 type, scope 使用英文，header, body 使用中文。

### 非技術背景人員（如本會秘書）

您不需要懂程式碼也能為本專案做出貢獻：
1.  **資料維護**：本系統的資料來源為指定的 Google Sheets。請在新增或修改議事資料時，嚴格遵守格式規範（基本上，第26屆以前地都可正常運作），避免因資料格式錯誤導致網頁解析失敗。
2.  **問題回報**：若您在使用網站時發現任何錯誤（如頁面破圖、資料未更新），請至 GitHub 的 `Issues` 區塊建立新工單，並盡可能提供截圖及發生時間。

## Pull Request 流程

1.  確保您的程式碼已經過本地測試，且能順利編譯。
2.  提交 Pull Request 時，請在描述欄位清楚說明此 PR 解決了什麼問題或新增了什麼功能。
3.  若該 PR 關聯到特定的 Issue，請在描述中使用 `Fixes #Issue編號` 的語法。
4.  懇請等待其他資訊人員或專案維護者的 Code Review，確認無誤後再行合併。
